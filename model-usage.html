📐 Optimization Rules
#1 Never Opus for data collection — 60× more expensive than Haiku for output tokens
#2 Never Haiku for strategy — cheap output that needs rework costs more than doing it right
#3 When in doubt, Sonnet — the safe middle ground
#4 Cascade up in multi-step workflows — start cheap, escalate for judgment
#5 QA always at Opus — catching errors at the quality gate saves rework
#6 Track cost per deliverable, not per token — a $2 Opus analysis that's right beats a $0.10 Haiku analysis that's wrong
